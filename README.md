# 九州绘灵卷（微信小游戏版）

本目录是将 nine 目录中的 HTML 版“九州绘灵卷”移植到微信小游戏运行时后的工程。

## 当前实现状态

- 已替换原始飞机模板入口，接入九州玩法主循环
- 已迁移并接线核心数据层：区域、精灵、技能、道具、奖励、图鉴
- 已迁移并接线核心逻辑：探索、战斗、捕捉、升级、多存档、精灵站
- 已支持完整主流程：开始页、存档管理、初始精灵选择、区域地图、探索、野战/道馆战、背包、商店、精灵管理、精灵站、奖励、图鉴
- 已接入素材化界面渲染（精灵/道具/馆主图片）和音效系统（BGM + SFX）

## 关键目录

```
├── audio/                     // 音频资源
├── images/                    // 图片资源
├── js/
│   ├── data/                  // 区域/精灵/技能/道具数据
│   ├── logic/                 // 战斗/捕捉/探索/升级/存档逻辑
│   ├── databus.js             // 全局状态容器
│   ├── main.js                // 微信小游戏主循环与 UI
│   └── render.js              // Canvas 与屏幕适配
├── game.js                    // 小游戏入口
├── game.json                  // 小游戏配置
└── project.config.json        // 微信开发者工具项目配置
```

## 运行方式

1. 打开微信开发者工具
2. 导入 jingling 目录
3. 编译运行（平台选择小游戏）

## 说明

- 当前 UI 为 Canvas 纯代码实现，优先保证微信小游戏编译与主流程可玩。
- 后续可继续将 nine 的更多界面细节（图鉴、奖励、商店、精灵站等）逐步补齐到当前框架中。
